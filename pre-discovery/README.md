# Pre-discovery: LLM endpoint discovery from APP_URL

Infers `TARGET_API_URL` from only `APP_URL` using Playwright. Captures network traffic (GET and POST), scores by LLM-likelihood heuristics, and writes `discovered_api.json` with comprehensive paths and payload formats.

Use when the LLM API endpoint is unknown — e.g. subdomain, 3rd party, or non-obvious internal path.

## Usage

### Full pipeline (recommended)

```bash
# From project root: prompts for site, then runs discovery → guide → script → run prompts
python main.py pre-discovery
```

Or run the pre-discovery module directly:

```bash
python -m pre-discovery.main
```

### Discovery only

```bash
# From project root (uses APP_URL from .config)
python -m pre-discovery

# With options
python -m pre-discovery --app-url http://localhost:3000 --update-config
```

### Options

| Option | Description |
|--------|-------------|
| `--app-url` | App URL to load (default: APP_URL from .config) |
| `--output-dir` | Where to write discovered_api.json (default: pre-discovery/format/) |
| `--update-config` | Update TARGET_API_URL in .config with discovered URL |
| `--headless` | Run browser headless |
| `--no-auto-trigger` | Skip auto-triggering chat; wait for manual message only |
| `--timeout` | Seconds to wait for first LLM request (default: 120) |
| `--verbose` | Log every GET/POST/WebSocket seen and its score (for debugging) |

## Cloudflare / Anti-bot Evasion

Pre-discovery applies multiple evasion techniques for Cloudflare-protected sites (e.g. chatgpt.com):

1. **Launch args** — `--disable-blink-features=AutomationControlled` and related flags to hide automation
2. **Real Chrome** — Uses installed Chrome (`channel="chrome"`) instead of bundled Chromium. Set `EVASION_USE_CHROMIUM=1` to fall back to Chromium
3. **Context fingerprint** — Rotated User-Agent, `locale`, `timezone_id` for realistic browser fingerprint
4. **Human-like behavior** — Longer delays before navigation, mouse movement and scroll warm-up after page load
5. **Remote browser (Technique 8)** — Use a managed browser service (Browserless, Scrappey) that handles Cloudflare:

   ```bash
   # Browserless (browserless.io) or similar
   export REMOTE_BROWSER_URL="wss://chrome.browserless.io?token=YOUR_TOKEN"
   # or
   export BROWSERLESS_URL="wss://chrome.browserless.io?token=YOUR_TOKEN"
   python -m pre-discovery --app-url https://chatgpt.com
   ```

   When set, Playwright connects to the remote browser instead of launching locally. These services use residential proxies and anti-detection infrastructure.

Optional: `PROXY_LIST=http://host:port` for proxy (e.g. residential proxy). `pip install playwright-stealth` for additional WAF evasion.

## Flow

1. Launches Playwright, loads app at APP_URL
2. If homepage (path `/`), tries navigating to `/chat`, `/ai-chat`, or `/conversation`
3. Optionally uses saved auth state (from prior `login`) if available
4. Attempts to auto-trigger chat (find textarea, type "hello", submit)
5. If not, prompts you to send one chat message manually
6. Captures GET and POST/WebSocket requests, scores by path and payload structure
7. Writes `discovered_api.json` with endpoints (paths and payload formats)
8. With `--update-config`, updates TARGET_API_URL in .config

## Output (format/)

- **discovered_api.json** — LLM endpoint discovery:
  - **target_api_url**, **method**, **confidence**, **score**, **reason** — primary LLM endpoint (backward compatible)
  - **candidates** — top POST/WS candidates
  - **endpoints.post** — POST/WebSocket endpoints with `url`, `path`, `method`, `score`, `reason`, `headers`, `payload_format` (fields, field_types, messages_structure)
  - **endpoints.get** — GET endpoints with `url`, `path`, `method`, `score`, `reason`, `query_params`
- **playwright/** — Playwright trace of the entire LLM interaction flow (screenshots, DOM snapshots, network), extracted for programmatic access. To view interactively, zip the dir and run `playwright show-trace playwright.zip`.
- **llm_api_guide.json** — definitive JSON guide for LLM GET/POST paths and formats (generated by `python -m pre-discovery.agents guide`). Set `PRE_DISCOVERY_LLM_TIMEOUT` (seconds, default 120) if guide generation hangs on slow API.
- **ask_capital_script.py** — Playwright script to ask questions via chat UI (generated by `python -m pre-discovery.agents playwright-script`). Loads prompts from `prompts/prompts.json` by default (array of objects with `prompt` key). Use `--no-headless` if Cloudflare blocks headless.
- **full_trace.json** — complete unabridged request trace (except image extensions):
  - **captured_at**, **app_url**, **request_count**
  - **requests** — every request with full `headers` (including cookie, authorization), `timestamp`, `elapsed_ms`, `url`, `path`, `method`, `query_params`, `post_data` (for POST), `response_status`. Image URLs (.jpg, .jpeg, .png, .svg, .ico, .webp) excluded.

## Integration

Component-discovery config automatically falls back to `pre-discovery/format/discovered_api.json` when TARGET_API_URL is not set in .config. So you can:

1. Run `python -m pre-discovery` (optionally with `--update-config`)
2. Run discovery/diagnostics as usual — TARGET_API_URL is resolved from pre-discovery output

## Heuristics

**POST/WebSocket** — scored by:
- **Payload**: `messages`, `prompt`, `model`, `userMessageContent`, `stream`, etc.
- **URL path**: `/chat`, `/completion`, `/api`, `/v1/`, `/v2/`
- **Origin**: same-origin bonus
- **Content-Type**: application/json

**GET** — scored by:
- **URL path**: `/conversation`, `/sessions`, `/models`, `/history`, `/api`, etc.
- **Origin**: same-origin bonus

Auth requests (grant_type, refresh_token) are excluded.

When the POST body is missing or small (e.g. cross-origin), scoring falls back to URL-only.
